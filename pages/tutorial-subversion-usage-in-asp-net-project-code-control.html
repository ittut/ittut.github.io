Versioning Software Systems: Introduction

Apache Subversion

Apache Subversion (commonly known as SVN, due to command line program name svn) is and revision control & software versioning system This is open source software. Developers use Subversion to keep current and history of source code files, web pages, and software documentation. Apache Subversion is mostly-compatible with its predecessor which is also wide spread, Concurrent Versions System. Subversion was developed by CollabNet Inc. The enterprises has also began to take on Subversion. Subversion was recognized as the leader in the Standalone Software Configuration (SCM) group and as a tough player in the Software Configuration and Change Management (SCCM) type.
The community of open sources are using Subversion extensively. Also Subversion hosting for projects that require versioning is provided by Google Code abd any other hostings like Assembla.

Revision Control System

The Revision Control System is a software that implements revision to achieve  automation for retrieval, logging, identifying, and merging of revisions and the most important storing data. Revision Control System is able of managing binary files, despite the fact that it is much less effective. Revisions are being stored by means of the diff software. Revision Control System is practical for text that is revised often, for example programs, documents, technical graphics.

Web Development Tips with Subversion

Commits as kind of atomic operations. Try to commit code changes that semantically are enclosed in a certain task. It will help in future find out what gone wrong in the code and when. Also it is very helpful to discard changes containing errors
