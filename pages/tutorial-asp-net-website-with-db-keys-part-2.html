ASP.NET Web Site with DB Keys: Part 2

20th December 2010    

In this tutorial I will explain how to create web page to edit data using database with relations and create data editing using data bound DropDownList control

Creating new page for edition

Create a new Web Form page and press Add:

Suppose its name is Default2.aspx by default. But actually it’s a page where we need to edit our records.

We will need two new data sources for each table. Open Server Explorer / Data Connections / Tables and drag and drop LinkCategories table onto the page. Then delete automatically created GridView1 control: we’ll need Data Source only. Repeat it for Links table.

Then switch to Design mode and drag and drop DetailsView control near SqlDataSource.

In DetailsView Tasks assign data source to SqlDataSource2 as shown below.

Configuring Data Source for edition

In SqlDataSource Tasks choose Configure Data Source, click Next. Press Where button. In open dialog fill values as shown on the image and press Add, then OK. Then Next and Finish.

Switch to Source Mode and Add DefaultMode="Edit" to the DetailsView control. Then add <asp:CommandField ShowEditButton="true" /> to Fields collection.

To perform editing, launch application and add to the URL ?ID=X, where X – is any existing record ID in the Links table. After refreshing page edit interface will appear.

Form capable to update date to the database will appear.

Note that LinkID field is Read-Only because its a primary key and may not change for specific record. To edit data in this form and assign right Link Category to link we need to remember Link Category ID for every category. This is not solution. Next part of tutorial deals with this poblem.
